# CI 遷移到 GitHub Actions：你在 7/13 前要做的事

**TL;DR**

- **這是什麼**：公司的建置系統要換掉。從 Jenkins 換成 GitHub Actions。
- **跟你什麼關係**：你的專案如果現在用 Jenkins 建置，就要遷移。沒遷移的話，之後會建置失敗。
- **你要做什麼**：3 步遷移（複製模板 → 改兩處設定 → 停用舊專案）。有用到密碼或金鑰的專案，還要多做一步。整體約 15 分鐘。
- **何時生效**：**2026-07-13（下週一）**新系統生效。**2026-07-20** 舊系統關閉。

## 術語表

新系統你可能沒用過，以下詞語全文會重複出現，先看過一遍：

| 詞語 | 意思 |
|---|---|
| CI（持續整合） | 你提交程式碼後，自動幫你建置與測試的系統，不用手動跑。 |
| Jenkins | 公司目前用的 CI 系統，本次要被取代。 |
| GitHub Actions | 新的 CI 系統，直接跑在 GitHub 上，設定檔跟著程式碼放在同一個倉庫。 |
| 工作流（workflow） | GitHub Actions 用來描述「怎麼建置你的專案」的設定檔，副檔名 `.yml`。 |
| 模板（template） | 已經寫好的工作流設定檔，複製後改幾個地方就能用在自己的專案。 |
| Secrets | 建置過程要用到、但不能公開的密碼或金鑰。 |

## 時間表

- **2026-07-13（下週一）**：GitHub Actions 正式生效，之後新的建置都走這裡。
- **2026-07-20**：Jenkins 關閉。這天之後，還沒遷移的專案會無法建置。

兩個日期之間有一週緩衝，讓還沒遷移的專案補做。

## 這次改了什麼

| 項目 | 此前（Jenkins） | 現在（GitHub Actions） |
|---|---|---|
| 建置設定放哪 | Jenkins 網頁上設定 | 倉庫內的工作流檔案（`.github/workflows/` 目錄下） |
| 平均建置時間 | 12 分鐘 | 7 分鐘 |
| 密碼／金鑰怎麼給 | 在 Jenkins 設定 | 要在 GitHub 重新設定，**不會自動搬過去** |

## 你要做什麼：3 步遷移

1. **複製模板**：把 `.github/workflows/template.yml` 複製一份，作為你專案的工作流檔案。
2. **改兩處**：把複製出來的檔案裡的專案名稱、建置指令，換成你自己專案的內容。
3. **停用舊專案**：到 Jenkins 你的專案頁面，點「停用」。

## 如果你的專案有用到密碼或金鑰

Jenkins 裡設定的密碼或金鑰，**不會自動搬到 GitHub Actions**。你需要手動補上，否則新的建置會因為缺少密碼而失敗：

- 進入你的 repo → **Settings → Actions secrets**
- 把原本在 Jenkins 設定的密碼／金鑰，逐一重新新增進去

如果你的專案完全沒用到密碼或金鑰，這一步可以跳過。

## 怎麼確認自己弄對了

- 觸發一次建置，能成功跑完（沒有因為缺設定或缺密碼而失敗）。
- 建置時間大致落在 7 分鐘上下（原本 Jenkins 是 12 分鐘）。
- 回到 Jenkins 頁面確認你的舊專案已顯示「停用」。

## 已知取捨

- 模板檔案內實際要改哪些欄位，本文未列出。請以模板檔案裡的說明為準。
