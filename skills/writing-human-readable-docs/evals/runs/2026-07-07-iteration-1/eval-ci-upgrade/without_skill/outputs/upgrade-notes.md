# CI 系統升級說明：Jenkins → GitHub Actions

## 這次改什麼

公司的 CI（自動構建流程）將從 Jenkins 遷移到 GitHub Actions，**2026-07-13（下週一）生效**。

Jenkins 會繼續保留一段時間讓大家轉移，但將於 **2026-07-20 正式關停**。也就是說，7/13 到 7/20 這一週是雙軌過渡期，7/20 之後所有專案都必須已經在 GitHub Actions 上跑。

## 為什麼要改

新的 CI 平均構建時間會從 **12 分鐘縮短到 7 分鐘**。

## 我需要做什麼（三步遷移）

每個專案負責人需要在 7/13 之後、7/20 之前完成以下三步：

1. **複製模板**：把 `.github/workflows/template.yml` 複製到自己專案的 workflow 設定裡。
2. **修改設定**：把模板裡的專案名稱與構建命令改成自己專案對應的內容。
3. **停用舊流程**：到舊的 Jenkins 專案頁面，點擊「停用」。

## 特別注意：Secrets 要手動搬

如果你的專案在 Jenkins 裡設定過 credentials（例如 API key、密碼、憑證等機密資訊），**這些不會自動搬到 GitHub Actions**。

你需要自己到該專案的 GitHub repo 頁面，走 **Settings → Actions secrets**，把需要的 secrets 重新輸入一次。

建議在做第 1、2 步（複製模板、改設定）的同時就把這件事一起做完，不要等到最後才發現構建因為少了 secrets 而失敗。

## 遇到問題找誰

遷移過程中有任何問題，到 **#ci-migration** 頻道發問即可。

## 時間軸總覽

|---|---|
