## 派工 Prompt（給最便宜模型）

用最便宜的模型跑批量檔案改寫是合理的，但 prompt 要寫得夠具體，避免它自由發揮。建議如下：

```
任務：更新 docs/ 目錄下所有 .md 檔案的 YAML frontmatter。

要做的兩件事：
1. 如果 frontmatter 沒有 updated 欄位，加上 updated: 2026-07-08（放在最後一行）。
2. 如果 tags 欄位是逗號分隔字串（如 tags: a, b, c），改成陣列格式：tags: [a, b, c]。

規則：
- 只修改 --- 與 --- 之間的 frontmatter 區塊，正文一律不動。
- tags 已經是陣列格式的檔案不要動。
- 沒有 frontmatter 的檔案跳過。
- 逐檔處理完後，輸出一份清單：每個檔案一行，標明「已改 / 跳過」。

完成標準：docs/ 下所有 .md 檔案都被檢查過，清單涵蓋全部檔案。
```

幾個建議：
- 批量任務給便宜模型跑之前，先拿 2-3 個檔案試跑，確認它改的格式是對的再放全量。
- 如果你的環境可以跑腳本，這種機械轉換其實用一個 python 腳本 + PyYAML 更省——一毛錢模型費都不用花，也不會有模型手滑改壞正文的風險。可以讓模型先幫你寫這個腳本，再人工過目一遍跑起來。
- 記得先 git commit 一次當還原點，改壞了可以直接 revert。
