# 分層矩陣：各層級該加什麼、該刪什麼、旋鈕在哪

蒸餾來源：Anthropic 官方 per-model 提示文檔（Claude Fable 5 / Opus 4.8 /
Sonnet 5）、OpenAI GPT 系提示指引、以及跨模型派工的實戰教訓。模型名會過時，
**層級特徵不會**——新模型出來先按 SKILL.md 的歸層探針落位，再套本表。

## Tier 1：Frontier-agentic（頂級旗艦）

代表能力類：Claude Fable/Mythos 5 級。特徵：多日級長自主執行、規格明確
問題的一次做對率高、可靠派遣並管理平行子代理、強指令記憶。

### 該給的（這層缺的是授權與邊界，不是能力）

- **意圖（why）**：說明請求背後的目的與受眾，模型會把任務連結到相關資訊
  而非自行腦補意圖。模板：`我在做 [更大的事] 給 [誰]，他們需要 [產出用途]。
  基於此：[請求]。`
- **邊界**：明確「什麼情況不要動手」。典型：用戶在描述問題／思考出聲時，
  交付物是評估不是修復；改變系統狀態的命令前先確認證據支持該動作。
- **停止條件／檢查點定義**：一句「只在真正需要用戶時停下：破壞性操作、
  真範圍變更、只有用戶能提供的輸入」勝過列舉每種情況。
- **證據錨定的進度聲明**：長自主執行必加——「報告進度前，把每條聲明對照
  本 session 的工具結果稽核；只報告有證據可指的工作，未驗證的明說」。
  實測這條幾乎完全消除虛構狀態報告。
- **自主權聲明**（headless 場景）：「你在自主運行，用戶不在線；可逆且
  符合原始請求的動作直接做；回合結束前檢查最後一段，若是計畫/提問/承諾，
  現在就用工具把它做掉」。防「純文字宣告意圖然後停住」的罕見早停。
- **子代理授權**：這層擅長派遣平行子代理，給出何時該委派的指引即可，
  鼓勵非阻塞（派出後繼續自己的工作）。
- **記憶錨點**：給一個寫筆記的地方（一個 markdown 檔就夠），說明「一檔
  一課、開頭一行摘要、不存 repo 已記錄的東西、錯了就刪」。

### 該刪的

- CoT 步驟清單、詳細推理腳手架——會壓低輸出品質。
- 逐條行為列舉——短指令引導大類行為已足夠。
- 強制進度節拍（「每 N 次工具呼叫總結一次」）——模型自帶良好節奏。
- **要求復述／展示內部推理**——可能觸發 reasoning-extraction 類 refusal，
  這是硬故障不是風格問題。要推理可見性走結構化 thinking 輸出或專用
  send-to-user 工具。
- 上下文餘量倒數顯示——會誘發「建議開新 session／自行縮減工作」。框架
  必須顯示時，補一句「你有充足上下文，不要因此停止或摘要」。

### 已知怪癖與對策

| 怪癖 | 對策提示 |
|---|---|
| 高檔位下對例行任務過度收集上下文、過度規劃 | 「有足夠資訊就動手；不要重推已確立的事實、不要複述用戶已做的決定」 |
| 順手做未要求的事（重構、防禦性備份） | 「不加超出任務的功能／抽象；bug 修復不需要周邊清理」 |
| 極長 session 尾部偶發早停（宣告意圖不執行） | 上面的自主權聲明；互動場景回「continue」即可 |
| 工作太久後最終訊息充滿自造縮寫與箭頭鏈 | 「最終總結面向沒看過過程的讀者：完整句子、術語展開、結論先行」 |

## Tier 2：主力（Strong workhorse）

代表能力類：Claude Opus 4.8 / Sonnet 5、GPT-5.x 主力檔。特徵：強代理
能力但指令解讀**字面化**——不自動把一個項目的指令推廣到另一個，不推斷
你沒提的請求。

### 該給的

- **明確範圍**：需要廣泛套用的指令要說明白（「套用到每個章節，不只
  第一個」）。字面化是特性不是缺陷——精確、可預測、少反覆，特別適合
  調好提示的管線。
- **工具使用時機**：這層傾向先推理後用工具；希望多用某工具就明確描述
  何時、為何、如何用它。升檔位（high/xhigh）本身也會顯著增加工具使用。
- **正面範例**：展示「怎樣算適當簡潔」的範例，比負面指令（「不要囉嗦」）
  有效。
- **review/分類類任務的覆蓋率聲明**：見 anti-patterns.md 的召回率殺手條目
  ——這層對「只報高嚴重性」會忠實執行到讓你以為模型變弱了。
- **風格重校**：換代後語氣基準會漂移（更直接、少肯定開場、少 emoji），
  產品依賴特定語氣就重新寫明。

### 該刪的

- **temperature / top_p 依賴**：新一代主力模型多數已不接受非預設採樣參數
  （會直接 400）。要多樣性改用「先提出 N 個方向讓用戶選再實作」。
- 強制中間狀態訊息——這層已自帶高品質進度更新，舊框架的強制節拍先移除
  再看效果。
- 為更弱模型寫的前端美學長提示——短片段（禁 generic 字體／紫漸變／
  模板化布局）+ 具體規格即可。

### 檔位語義（重要：檔位名跨代不等值）

跨代遷移按**觀察到的思考長度**對齊，不按檔位名。經驗對照：新一代的
medium ≈ 上一代的 high。低檔位下模型會嚴格框定在被要求的範圍內——這對
成本好，但中等複雜任務有思考不足風險；發現推理太淺，第一手段是升檔，
不是改提示。

## Tier 3：快速經濟（Fast economy）

代表能力類：Haiku 級、GLM 等開源主流檔、各家 mini/flash。特徵：便宜、
快，但模糊容忍度低、自我驗證不可信、格式漂移常見。

### 該給的（經典提示工程在這層依然全額有效）

- **步驟清單**：把「怎麼做」寫成編號步驟，一步一動作。
- **輸出格式契約**：精確到欄位名與順序；能用 JSON schema／結構化輸出
  就用，不要靠散文描述格式。
- **範例（few-shot）**：至少一個完整的輸入→輸出範例；格式敏感的任務給
  2-3 個覆蓋邊界情況。
- **XML／區塊結構**：用穩定的標籤分隔指令、輸入、範例、輸出要求——
  這層對結構的依賴是真實的。
- **明確停止條件**：「完成 X 就停，不要繼續做 Y」；不寫就會過度生成。
- **驗證外置**：不要指望它自查。驗收由外部承擔——測試、上層 agent、
  獨立 verifier、或程式化斷言。

### 該刪的

- 開放式判斷委託（「你自行決定最佳方案」）——會拿到淺層第一直覺。
- 大段背景敘事——這層上下文利用率低，只給任務直接需要的上下文。
- 對「意圖理解」的依賴——why 可以寫（一句），但不能替代 how。

## 跨供應商旋鈕對照

| 旋鈕 | Claude 系 | GPT 系 | 開源（GLM 等） |
|---|---|---|---|
| 智慧/成本主旋鈕 | `effort`（low→max） | reasoning effort | thinking 開關／部署方檔位 |
| 思考控制 | adaptive thinking（部分型號預設開） | reasoning 模式選擇 | `<think>` 開關或專用推理版模型 |
| 冗長度 | 提示引導 | `verbosity` 參數 + 提示 | 提示引導 |
| 採樣多樣性 | 新主力不接受 temperature；用「提案→選擇」模式 | 依型號 | 通常可用 temperature |
| 結構化輸出 | tool use / 結構化輸出 | structured outputs | 依部署（vLLM guided decoding 等） |

GPT 系派工的區塊化提示骨架（`<task>` / 輸出契約 / 跟進政策 / 驗證迴路 /
接地規則）見 delegation-prompts.md 的跨供應商段。

## 維護原則

新模型發布 → 跑歸層探針 → 只在「層級特徵與本表不符」時改本表；
單一模型的怪癖記到「已知怪癖」表，不要為單一模型另立層級。

**本表 = 層級不變式；[model-roster.md](model-roster.md) = 具體模型快照。**
「哪個真實模型現在落哪層」屬於會過時的快照，寫進 roster（帶日期、可拋棄）；
本表只維護跨代穩定的層級特徵與旋鈕語義，不列具體型號的當期落位。
