# 改寫協議：層級感知的提示優化流程

經典提示改進器的流程（結構化 → 加 CoT → 增強範例）只有一個方向：加。
本協議先判方向再動手——**升層遷移主要是刪，降層委派主要是加，同層優化
才是調**。適用對象：skill 文本、產品 system prompt、review prompt、
派工模板、任何會被重複使用的提示。

## 第一步：判定（30 秒，不可跳）

| 問題 | 答案決定什麼 |
|---|---|
| 目標模型是誰、哪一層？ | 加還是拆（SKILL.md 三層速查） |
| 執行面？（互動 / headless / 產品模板） | 完成定義與輸出契約的厚度 |
| 提示原本為哪個模型寫的？ | 升層遷移 or 降層委派 or 同層 |
| 有沒有延遲/成本硬約束？ | 檔位選擇；鎖低檔才允許提示補償 |

## 第二步：診斷（症狀 → 旋鈕）

先確認問題該用參數修還是提示修：

| 症狀 | 第一手段 | 提示手段（第二） |
|---|---|---|
| 推理淺、漏步驟 | 升 effort/reasoning 檔位 | 鎖低檔時加「此任務涉及多步推理」 |
| 過度思考、太慢 | 降檔位 | 「有足夠資訊就動手」 |
| 太囉嗦 | — | 正面簡潔範例 +「選擇性收錄，不是壓縮寫法」 |
| 不用工具 | 升檔位 | 明確描述工具的何時/為何/如何 |
| 指令沒推廣到全部對象 | — | 明確範圍（「每個章節，不只第一個」） |
| review 召回率降 | — | 覆蓋率聲明（anti-patterns #2） |
| 虛報進度/完成 | — | 證據錨定聲明（tier-matrix Tier 1） |
| 過度工程、順手重構 | — | 邊界聲明（「不加超出任務的抽象」） |
| 風格漂移 | — | 按新基準重寫風格提示（先裸跑再補差） |

## 第三步：改寫

### 方向 A：升層遷移（舊提示 → 更強模型）——預設動作是刪

逐條過舊提示，每條問「新模型不看這條會做錯嗎」，答不上來就刪。
固定審計清單：

1. ☐ 刪 CoT 步驟腳手架（anti-patterns #5）
2. ☐ 刪「復述推理」類指令（#1，refusal 風險）
3. ☐ 刪強制進度節拍（#3）
4. ☐ 刪 temperature 依賴，改提案-選擇模式（#7）
5. ☐ 「只報重要的」類過濾改成覆蓋率+下游過濾（#2）
6. ☐ MUST/NEVER 堆疊改寫成 why（#4）
7. ☐ 風格補丁重新答辯（#13）
8. ☐ 補上這層真正缺的：意圖、邊界、停止條件、證據錨定（tier-matrix Tier 1）

刪完跑一次真實案例：新版本 vs 舊版本各跑同一組任務，比輸出。刪錯了
（行為真的退化）再把那一條加回來——這比憑感覺保留全部便宜得多。

### 方向 B：降層委派（為更弱的執行者加厚）

按 delegation-prompts.md 的經濟層模板加厚：步驟清單、格式契約、範例、
前置分支判斷、明確停止條件。重點是**把你腦中的隱含判斷顯式化**——你
知道「格式不符就跳過」，執行者不知道。

### 方向 C：同層優化（產品模板迭代）

模板 + 變數是基礎設施：固定部分與動態部分（用戶輸入、檢索內容、
歷史）分離，變數用穩定的定界（XML 標籤包裹 `{{變數}}`），版本控制只
追蹤固定部分。經濟層模板的結構增強（清晰分段、範例前置、輸出格式
錨定）仍然全額有效——那套經典方法論本來就是為這層設計的。

## 第四步：驗證（沒有這步不算優化）

- **有 eval**：改動前後各跑一輪，看指標不看感覺。review/分類類看
  召回率與精確率；生成類至少做盲評對比。
- **沒 eval**：拿 3-5 個真實案例 A/B。連這個都沒做的提示改動，
  等於沒改——你不知道方向對不對。
- **產品 system prompt**：改動必須配回歸集；一字之差在百萬次調用上
  就是可測的行為漂移。

## 常見誤判

- **「新模型效果變差了」**：先查是不是框架效應——舊提示的過濾指令被
  更忠實地執行（召回率殺手）、舊腳手架壓制了新能力。裸跑一次再下結論。
- **「加了規則還是不聽」**：查層級——對經濟層，散文規則不如結構化
  契約；對 frontier 層，可能是規則之間互相矛盾，模型在遵守你沒注意到
  的另一條。
- **「同樣的提示有時好有時壞」**：檔位沒鎖、或執行面變了（互動下
  可反問的模糊，headless 下就是缺陷）。
