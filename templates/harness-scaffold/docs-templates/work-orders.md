# Harness 工單登記處

> 每張工單自帶：目標、設計決策（已定案，執行者不需重新設計）、驗收標準、測試要求。
> 執行規則：一張工單 = 一個獨立 session，完成 = 驗收標準全過 + 可信集綠 + commit。
> 執行者遇到規格與現實衝突時：停下來登記，不要自行改設計。
> status 收帳：`done(<證據：檔案路徑/commit>)`，不是「已完成」三個字。
